# Vilm Lyrics Aligner 간단 사용법

## 데스크톱 앱

1. 편집본과 일치하는 영상 또는 음원 파일을 엽니다. 여러 클립으로 구성된 복잡한 NLE 타임라인이라면 가벼운 전체 타임라인 참조 파일을 먼저 출력합니다.
2. 전체 가사를 붙여 넣거나 UTF-8 TXT 파일을 불러옵니다.
3. **전체 파일**을 사용하거나, 파형에서 필요한 구간을 드래그한 뒤 **선택 구간**을 사용합니다.
4. 대부분의 곡은 **자동 모드** 그대로 두고 **SRT 생성**을 누릅니다.
5. 생성된 SRT를 편집 프로그램에 불러옵니다. Desktop SRT에는 0.5초 길이의 0초 기준용 비표시 자막이 자동으로 들어가므로, 자막 파일을 타임라인 시작점에 놓으면 첫 실제 가사 전의 시간 간격도 유지됩니다.

자막 시간값을 편집 타임라인과 맞춰야 한다면 전체 타임라인 참조 파일을 사용하세요. 구간만 따로 출력한 파일은 원래 타임라인 시작 위치 정보가 없으면 보통 `00:00:00`부터 시작합니다.

## DaVinci Resolve Studio

1. 대상 타임라인을 엽니다.
2. `Workspace > Workflow Integrations > Vilm Lyrics Aligner`를 엽니다.
3. 전체 가사를 붙여 넣고 **자막 생성**을 누릅니다.

일부만 처리할 때는 타임라인에 In과 Out을 모두 설정하고 **In/Out 구간**을 선택합니다. 전체 가사를 넣어도 선택한 오디오에서 들리는 연속된 가사 범위를 찾아 그 부분만 정렬합니다. 기존 자막은 삭제하지 않습니다.

직접 연동 패널은 Windows 또는 macOS용 Resolve Studio가 필요합니다. Resolve 무료 버전에서는 데스크톱 앱으로 SRT를 만든 뒤 가져올 수 있습니다.

설치할 때 Resolve 연동을 선택하면 Python.org 3.12가 있는지 확인하고, Resolve가 사용할 수 있는 설치가 없을 때만 동봉된 공식 패키지를 설치합니다. 이 공용 스크립팅 런타임은 앱의 AI 전용 Python과 별개이며 다른 Resolve 스크립트도 사용할 수 있으므로 Vilm 제거 후에도 유지됩니다.

## 자동 모드와 수동(고급) 모드

- **자동 모드**는 줄바꿈을 강한 힌트로 사용하지만, 지나치게 길거나 느린 줄을 나누고 명백한 표시용 줄바꿈만 합칠 수 있습니다.
- **수동(고급) 모드**는 비어 있지 않은 가사 한 줄을 자막 한 개로 유지하며 최대 글자 수, 최대 표시 시간, 끝 여운을 조절할 수 있습니다.

수동 모드에서 자막으로 보이지 않을 허밍은 별도 괄호 줄로 적을 수 있습니다.

```text
첫 번째 가사
(음)
다음 가사
```

`(음)`, `(흐음)`, `(oh)`처럼 실제 소리에 가까운 짧은 표현을 사용합니다. 일반 가사 줄 안의 괄호는 그대로 표시됩니다.

## 처리 장치

Windows의 검증된 NVIDIA 환경에서는 CUDA를 사용합니다. Apple Silicon Mac에서는 Whisper 정렬에 Apple Metal을 사용하고 보컬 분리는 CPU로 처리합니다. 두 플랫폼 모두 CPU를 문제 해결용 대체 경로로 선택할 수 있지만 훨씬 느립니다.

## 파일과 개인정보

모든 처리는 로컬에서 진행됩니다. 임시 디코딩 음원과 분리된 보컬은 완료 또는 실패 후 삭제됩니다. 데스크톱 SRT는 사용자가 정한 위치에, Resolve SRT는 `문서\Lyrics Aligner`에 저장됩니다. 진단 JSON은 로컬에 최대 30일 보관되며 가사를 바꾸거나 외부로 업로드하지 않습니다.
