# Vilm Lyrics Aligner 로드맵

기준 버전: `1.0.0` (2026-07-14)

## 지금 된 것

- Resolve Studio 현재 타임라인 또는 In/Out 범위를 임시 미디어로 렌더한다.
- Demucs 보컬 분리 후 Whisper `small`로 다국어·코드스위칭 공연을 분석한다.
- 가사 원문의 줄바꿈을 큐 경계로 우선하며, 출력 문구는 항상 원문을 사용한다.
- ASR 비교로 일부 변경·누락·반복과 실제 공연 순서 차이를 진단한다.
- 신뢰할 수 있는 ASR 시간대를 로컬 강제 정렬 앵커로 사용하고, 약한 구간은
  전역 정렬로 폴백한다.
- 보컬 활성도를 이용해 시작점과 지속음 꼬리를 보정한다.
- 다음 소절과 겹치지 않는 끝 유지와 기본 10초 최대 표시 시간을 적용한다.
- 긴 인터루드에 다음 가사가 미리 들어오던 실전 샘플 오류를 수정했다.
- 생성된 SRT를 현재 타임라인 자막 트랙으로 자동 삽입한다.
- 정상 종료 시 렌더 파일, WAV와 보컬 stem 등 임시 작업물을 삭제한다.
- Windows에서 CUDA 가능 여부를 자동 검증하고, 불가능하면 CPU로 동작한다.
- Python이 없는 PC용 단일 EXE 부트스트랩 설치기와 제거 기능을 구현했다.
- 설치기가 내려받는 uv 버전과 ZIP SHA-256을 고정하고 추출 전에 검증한다.
- `base`와 `small`을 같은 실전 샘플로 비교해 품질이 나은 `small`로 고정했다.
- 한국어/영어 UI 전환과 내부 음성 언어 자동 감지를 추가했다.
- 한국어/영어 사용자 가이드와 처리 구조·장치 정책을 문서화했다.
- In/Out 마크가 없으면 전체 타임라인으로 폴백하지 않고 안내한다.
- Resolve 내장 Python에서는 렌더와 정렬을 검증된 동기 흐름으로 실행한다. 장기적으로 별도 앱 프로세스로 옮겨 응답성과 취소를 제공한다.
- 24시간 이상 된 임시 작업 폴더와 30일 이상 된 진단 JSON을 자동 정리한다.

- 편집기와 독립적으로 실행되는 Windows/Apple Silicon macOS Desktop 앱을 구현했다.
- 영상·음원 파형에서 일부 구간을 선택하고 해당 구간만 디코딩·정렬할 수 있다.
- 선택 구간의 원본 시간 오프셋을 SRT에 보존한다.
- 설치기에서 Desktop은 기본 설치하고 Resolve Studio 연동은 감지 시 선택 설치한다.
- Windows 시작 메뉴/제거 항목과 macOS 전용 앱 데이터·제거 스크립트를 구성했다.
- macOS 첫 실행 전용 런타임 설치, Whisper MPS/Metal 검증, CPU 폴백과 Resolve Studio 선택 설치를 구현했다.
- Apple Silicon `.app`/DMG 빌드와 Developer ID 서명·공증 경로를 구현했으나 실제 M1 Pro 검증은 남아 있다.

## 현재 품질 기준선

- 한 큐 기본 최대 글자 수: 30자
- 시작 선행: 약 100 ms
- 발성 뒤 기본 유지: 약 1.2초
- 한 큐 기본 최대 표시: 10초
- 다음 발성이 가까우면 이전 큐 끝을 자동 단축
- 원문과 ASR이 다르면 원문 유지 + 진단 JSON 기록

이 값들은 완전한 미학적 정답이 아니다. 노래 자막은 호흡과 편집 연출의 영향을
받으므로, 목표는 무수정 자동화보다 수동 미세 조정량을 크게 줄이는 것이다.

## 다음 작업

### P0 — 현재 PC 실기 검증

1. Resolve에서 한국어/영어 UI 전환과 설정 저장을 확인한다.
2. 전체 타임라인과 In/Out 구간에서 SRT 생성·삽입을 확인한다.
3. 전체/In-Out 렌더와 동기 정렬 완료를 실제 Resolve에서 확인한다.

### P0 — 설치판 실기 검증

1. 깨끗한 Windows 11 VM의 Python 미설치·NVIDIA 없음 환경에서 설치, CPU
   런타임/모델 다운로드, Resolve 실행, SRT 생성·삽입과 제거를 확인한다.
2. 깨끗한 NVIDIA Windows PC/VM에서 같은 흐름과 CUDA 실제 사용을 확인한다.
3. 설치 중 네트워크 끊김, 재설치, 설치 취소, Resolve 실행 중 제거의 복구 동작을
   확인한다.
4. 설치 로그에 비밀정보나 불필요한 사용자 경로가 남지 않는지 점검한다.

### P0 — Apple Silicon Mac 실기 검증

1. M1 Pro 16GB에서 ad-hoc DMG 빌드, 첫 실행 전용 런타임 설치와 `mps` 검증을 확인한다.
2. 전체 파일·파형 선택 구간·CPU 폴백·취소가 Windows 결과와 구조적으로 같은지 확인한다.
3. Resolve Studio 패널 설치, 전체 타임라인과 In/Out 자동 배치, 마크 복원을 확인한다.
4. 앱/모델/패널 제거 후 시스템 Python과 다른 프로젝트 환경이 그대로인지 확인한다.

### P1 — 설치 경험 마감

- 설치 전 예상 다운로드/설치 용량과 CPU 모드의 예상 속도를 표시한다.
- NVIDIA가 감지되면 GPU 가속을 권장 기본값으로 사용하되 CPU 전용 선택을
  보조 옵션으로 제공한다.
- 실패 시 사용자가 이해할 수 있는 요약과 진단 로그 열기 버튼을 제공한다.
- 업데이트/다운그레이드와 설정 마이그레이션 정책을 정한다.

### P1 — 실사용 피드백 루프

- 서로 다른 가수·반주·언어 혼용 곡 5~10개에서 시작/끝 오차와 수동 수정량을
  기록한다.
- 여러 곡에서 실제 수동 수정량을 기록하고, 자동 로직은 검증 자료가 있을 때만 조정한다.

### P2 — 플랫폼 안정화 뒤

1. 제한된 지원 카드에서 Windows ROCm 실험 여부 결정
2. 다중 버전 업데이트/마이그레이션
3. 원문 없는 일반 영상 자동 자막 모드

## 의도적으로 하지 않는 것

- 사용자가 내부 모델이나 CUDA/CPU 백엔드를 Resolve 패널에서 직접 고르게 하지 않는다.
- ASR 문장을 사용자 가사 대신 자동으로 삽입하지 않는다.
- 설치 크기만 줄이려고 `base`로 낮추지 않는다.
- 검증 자료 없이 자동 타이밍 휴리스틱을 추가하지 않는다.
- Windows/macOS 검증이 끝나기 전에 원문 없는 모드로 범위를 넓히지 않는다.
