# Vilm 디자인 시스템 — 참고 명세

> **목적**: Vilm Care(Windows·WPF/.NET)의 실제 디자인 토큰을 다른 Vilm 브랜드 앱(파이썬 GUI)에서 재현하기 위한 단일 참고. 색·폰트·간격·톤은 프레임워크 무관하게 hex·px·원칙으로 기술하고, 문서 끝에 tkinter / PyQt(PySide) / customtkinter 매핑 힌트를 둔다.
> **출처**: `VilmCare.App/Themes/{Colors,Typography,Buttons,Cards,Tabs,Dialogs}.xaml` + `Controls/StatusCard.xaml` 실코드에서 추출(추측 아님).

## 1. 브랜드 무드 (한 줄로)

**"따뜻한 크림 종이 위, 진초록 액센트의 각진 미니멀 유틸리티 — 정보는 빽빽하되 톤은 차분하고 존중하는."**

- **정체성**: 크리에이터(음악·영상·스트리밍)용 시스템 도구. 신뢰·차분·따뜻함. 화려함이 아니라 "믿음직한 도구".
- **5원칙** (모든 디자인·문구 결정의 기준): **쉬움 · 엄밀 · 안전 · 가벼움 · 표준**.
- **DO**: 크림/베이지 배경, 매트한 플랫 면, 각진 형태(코너 거의 0), 높은 정보 밀도, 존댓말 톤.
- **DON'T**: 네온·형광색, 큰 둥근 카드, 무거운 그림자·그라데이션, 다크모드 기본, 과장된 애니메이션.

## 2. 색상 팔레트 (전체 토큰)

### 2-1. 표면 · 텍스트 · 구조

| 역할 | HEX | 설명 |
|---|---|---|
| 앱 배경 | `#F4EFE0` | 따뜻한 크림/베이지. 창 기본 바탕 |
| 카드 배경 | `#FAF8F2` | 배경보다 살짝 밝은 크림 |
| 스마트카드 배경 | `#F4F3E7` | 해석/지능 기능 카드용(미세하게 다른 톤) |
| 타이틀바 | `#E8E0CF` | 연베이지 |
| 버튼 면 | `#EDE8DA` | 크림 |
| 본문 텍스트 | `#1A1A1A` | 거의 검정(순검정 아님 — 부드럽게) |
| 보조 텍스트 | `#7A6B55` | 따뜻한 회갈색. 설명·부가정보 |
| 구분선·테두리 | `#C4B69C` | 탠(tan). 카드·버튼 외곽 |
| 스마트카드 테두리 | `#3E5A2E` | 잎초록. 지능 기능 강조 |
| 푸터 배경 | `#043D20` | 진초록(액센트가 면으로) |
| 푸터 텍스트 | `#F4EFE0` | 크림(진초록 위 반전) |

### 2-2. 의미색 시스템 ★ (상태 표현 규칙 — 가장 중요)

| 의미 | HEX | 쓰임 |
|---|---|---|
| **양호 / 정상 / 평소** | `#043D20` (진초록) | 기본 긍정 상태. 브랜드 대표색 |
| **주의 (살펴볼 것)** | `#F6C522` (앰버/브랜드 옐로) | 경미한 경고, 도움말 "?" 표식, 알림 배지 |
| **위험 / 경고** | `#BB2222` (레드) | 실제 문제. 굵은 상태 텍스트 |
| **중립 (의미 없는 수치)** | `#925D1C` (브라운) | 게이지·보조 지표처럼 좋고나쁨 없는 값 |

**규칙**:
- 색이 곧 판정이다. 좋은 상태는 초록, 나쁜 상태는 빨강 — **섞지 않는다**.
- **거짓 안심 금지**: 나쁜 상태를 초록/"괜찮음"으로 칠하지 않는다.
- **공포 조장 금지**: 애매하면 빨강 대신 앰버(주의) + 보수적 문구.
- 앱 아이콘도 이 논리 위: 평소 = 크림 배경·초록 마크, 알림 = **색 반전**(초록 배경·크림 마크)으로 명도를 통째 뒤집어 눈에 띄게.

## 3. 타이포그래피

- **본문 폰트**: `Segoe UI` (Windows 기본 산세리프)
- **고정폭 폰트**: `Consolas` (수치·로그·모노스페이스)

| 스타일 | 크기(px) | 굵기 | 색 |
|---|---|---|---|
| 헤더(창 제목) | 15 | Bold | 본문색 |
| 섹션 제목 | 13 | Bold | 본문색 |
| 소제목/라벨 | 11 | Bold | 본문색 |
| 본문 | 11 | Regular | 보조색 `#7A6B55` |
| 작은 글씨 | 10 | Regular | 보조색 |
| 푸터 | 12 | Regular | 크림(진초록 위) |
| 기본(창 전역) | 11 | Regular | 본문색 |
| 상태(양호) | — | Bold | 초록 `#043D20` |
| 상태(경고) | — | Bold | 빨강 `#BB2222` |

- **계층 원칙**: 제목은 진한 본문색(`#1A1A1A`) + Bold, 본문·설명은 흐린 보조색(`#7A6B55`). 강조는 색이 아니라 굵기+의미색.
- **작은 폰트(10~15px)** = 정보 밀도 높은 유틸리티 성격. 여백보다 내용.
- ⚠ 파이썬 tkinter는 폰트 크기가 pt 단위 → px ÷ 1.33 대략 환산(11px ≈ 8~9pt). PyQt/customtkinter는 px 그대로 가능.

## 4. 간격 · 형태

- **코너 반경**: **2px** (거의 각짐). 클래식·견고한 유틸리티 느낌. 큰 라운딩 지양.
- **테두리**: 카드 **2px**, 컨테이너 1px, 색 `#C4B69C`(탠). 스마트카드만 잎초록 2px.
- **카드**: 바깥 여백(margin) `8,4`(좌우8·상하4), 안쪽 여백(padding) `10,8`.
- **버튼 패딩**: 기본 `12,4` · 보조 `10,2` · 소형 `4,0` · 상세 `12,2`.
- **그림자**: 거의 없음(툴팁만 약한 드롭섀도). 전반적으로 **플랫·매트**.
- **줄 간격**: 카드 내 블록 사이 `4~8px`(상세 텍스트 위 4, 버튼줄 위 8).

## 5. 컴포넌트 명세

### 5-1. 카드 (핵심 UI 단위)
GroupBox 형태 — 헤더(제목, Bold) 달린 박스. 내부 구조:
```
┌ [제목] ───────────────────────────┐
│ 설명 텍스트(좌)          상태(우, 굵게·의미색) │
│ 상세/부연 텍스트(보조색)                      │
│ [조치버튼] [조치버튼]              [ ? ](우하단) │
└──────────────────────────────────┘
```
- **일반 카드**: 배경 `#FAF8F2` + 테두리 탠 2px.
- **스마트 카드**(해석·지능 기능 = 이 앱의 차별점): 배경 `#F4F3E7` + **잎초록 테두리 `#3E5A2E`**. 구조는 동일, 색만 달라 "특별한 기능"임을 시각적으로 구분.
- 상태 텍스트는 **우측 정렬 + 굵게 + 의미색**.

### 5-2. 버튼
- 크림 면(`#EDE8DA`) + 탠 테두리(2px) + 검정 텍스트. 플랫·각짐.
- 크기 계층: **Action**(최소너비 84) · **Secondary**(56) · **Small**(32, Bold, 테두리 1px) · **Detail**(60).

### 5-3. 도움말 "?"
- **앰버(`#F6C522`) 원, 지름 16px**, 테두리 1.2px, 내부 "?". 카드 우하단 배치.
- Hover 시 반투명(opacity 0.5). 클릭이 아니라 **툴팁으로 설명**을 띄우는 용도.

### 5-4. 툴팁
- 배경 `#F4EFE0` + 테두리 1px 탠 + 약한 그림자. **최대너비 340px**, 11px, 자동 줄바꿈. 여백 `10,8`.

### 5-5. 탭
- 탭 아이템: 크림 면(`#EDE8DA`), 패딩 `8,3`. 탭 컨트롤 배경은 앱 배경색.

### 5-6. 다이얼로그(별창)
- 배경 `#F4EFE0`, Segoe UI, 기본 12px. 본창과 같은 팔레트.

## 6. UX 라이팅 (문구 톤 — 디자인만큼 중요)

- **한국어 존댓말 "~해요"체**가 기본. 부드럽고 신뢰감. (영어 토글 시 대응)
- **단정 금지**: "~입니다" 대신 "~일 수 있어요"(보수적, 특히 진단·경고).
- **거짓 안심 금지 & 공포 조장 금지**: 나쁜 상태를 "괜찮아요"로 덮지 않되, 겁주지도 않는다. 중간의 "해석" 톤.
- **정보성 수치 = 전보체**("CPU 평소 수준(20%)"), 경고가 실리면 문장체("조금 빠듯한 편이에요").
- 짧고 명료하게. 전문용어는 최소화하고 쓰면 풀어서.

## 7. 파이썬 구현 힌트

| 프레임워크 | 적합도 | 재현 방법 |
|---|---|---|
| **PyQt / PySide (QSS)** | ★★★ 가장 정밀 | QSS 스타일시트에 hex·border·padding 그대로. `background:#FAF8F2; border:2px solid #C4B69C;` 식. 폰트 px 지정 가능 |
| **customtkinter** | ★★ 모던·간편 | `fg_color`/`border_color`에 hex. 단 기본이 둥근 코너 → `corner_radius=2`로 각지게. 다크모드 기본이니 `set_appearance_mode("light")` + 색 수동 지정 |
| **tkinter(ttk)** | ★★ 순정 | `ttk.Style().configure()`로 색·폰트. 테두리는 `relief="solid", borderwidth=2`. 폰트는 `("Segoe UI", 9)`, 고정폭 `("Consolas", 9)`. px→pt 환산 필요 |

**공통 시작점**:
- 창 배경 `#F4EFE0`, 카드 프레임 `#FAF8F2` + 탠 테두리, 본문 `#1A1A1A`/설명 `#7A6B55`.
- 상태 라벨은 굵게 + 의미색(초록/앰버/빨강/갈색).
- 폰트 Segoe UI(없으면 시스템 산세리프), 수치는 Consolas.
- 코너는 최대한 각지게(2px 또는 0), 그림자 없이 플랫.
